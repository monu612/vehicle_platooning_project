## 2025-02-28 - Precomputing NetworkX Simple Paths on Dynamic Graphs
**Learning:** Calling `nx.all_simple_paths` during a hot simulation loop on a dynamically changing graph is a massive performance bottleneck. Because the nodes in the topology are static and only the edges fail, standard NetworkX pathfinding recalculates the entire tree over and over.
**Action:** Precompute the simple paths once on the initial pristine static base topology (which contains all possible links) before any mutations occur. During the simulation loop, use the precomputed paths and filter them on the fly using `G.has_edge(u, v)` to exclude paths that are currently broken. This turns expensive pathfinding into a simple iteration, significantly cutting down simulation time (observed >2x speedup).
